Mandolin

slices up Gherkin to make it more versatile

Download this project as a .zip file Download this project as a tar.gz file

What is mandolin?

mandolin is a utility for slicing up Gherkin files which are used as part of Behavior Driven Development.

Why?

Gherkin is great but one of its main strengths, having a simple clear editable-by-anyone format, can also present issues with scalability and maintenance when reusing the scenarios for regression testing.

By providing a way to strip out and then re-merge tables to and from separate pipe delimited text files, the data can be managed in any way you like

mandolin can do the following:

  • strip out tables and place each one in a pipe separated file
  • merge a feature file (containing table placeholders) with a set of pipe separated data files

How?

mandolin is a short script written in perl which takes arguments telling it what to do. The reason for choosing perl (which admittedly doesn't feature large in BDD circles) is to have something that will work with any tooset, whether it be JBehave, Cucumber, Behat, Lettuce etc.

  • to extract tables as pipe delimited test files: mandolin.pl -e name_of_feature_file.feature
  • it will then create a directory and place a new feature file with placeholders for tables in the form of: '<>' and a series of text files each one containing a corresponding table. This may be very useful or very annoying if all you want to do is extract one or two tables. In that case, just extract the tables manually using a well-formed placeholder then perform a merge
  • to merge a feature file with one or more tables: mandolin.pl -m name_of_feature_file.feature

Does it do anything else?

No - but it may be useful to have a way of managing locators. Page Objects and other paradigms for creating and managing an imprint of the system under test doesn't really have a representation in Gherkin. Perhaps it would be nice if mandolin substituted locater placeholders with values held in a corresponding repository. When the system under test changes, the corresponding entry in the repository would be changed rather.

Can I have a script in something other than perl?
